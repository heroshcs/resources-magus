Manual de instalação

Prep Call Agent
em menos de 10 min

Um agente de IA que lê sua agenda às 18h e envia um briefing completo antes de cada reunião com participante externo. Tudo automático.

⚡ Setup em 10 min 📬 Email automático às 18h 🔍 Deep Search por reunião

Resultado

O que você vai receber

Para cada reunião com participante externo no dia seguinte, você recebe um email com:

  • Situação atual da empresa — baseada em fontes reais da internet
  • O que o cargo do participante provavelmente prioriza
  • 3 hipóteses de dor específicas para aquela empresa naquele momento
  • Uma observação de abertura baseada em sinal real — sem pitch genérico
  • 3 perguntas de Discovery prontas para usar
✦ Tudo pronto antes da reunião, sem você fazer nada. O agente roda todo dia às 18h automaticamente.

Verificando o Node.js

O que é o Node.js?
Imagine que o Claude Code é um aplicativo que precisa ser "executado" no seu computador, como um programa. O Node.js é o motor que permite rodar esse tipo de programa. Pense nele como a "tomada" — sem ela, o aparelho não liga, mesmo que o aparelho seja perfeito.
1
Abra o terminal de comandos do seu computador.
Digite CMD no campo de busca do Windows ou macOS.
2
Na tela do terminal, digite e aperte Enter:
Terminal
node --version
3
Se aparecer um número como v18.x.x ou superior — está correto, prossiga.
Se não aparecer: acesse nodejs.org/en/download, baixe e instale. Leva menos de 2 minutos.

Gerar sua Gmail App Password

A App Password é uma senha de 16 caracteres exclusiva para esse agente. Ela permite que o agente envie e-mails pelo seu Gmail sem expor a sua senha principal.

1
Acesse: myaccount.google.com/security
2
Confirme que Verificação em 2 etapas está Ativa — se não estiver, ative antes de continuar
3
Na mesma página, pesquise por App Passwords e clique no resultado
4
Clique em "Select app" → escolha Mail
5
Clique em "Select device" → escolha Windows Computer
6
Clique em "Generate"
7
Copie os 16 caracteres exibidos (ex: abcd efgh ijkl mnop) — você vai usar no próximo passo
💡
Guarde essa senha — você vai precisar dela no Passo 4. Se perder, basta gerar uma nova.
🔒
É segura — essa senha dá acesso apenas ao envio de email por esse app específico, não à sua conta Google completa. Pode ser revogada a qualquer momento.

Passo 4
Rodar o Setup Automático do Agente

Copie o prompt abaixo e cole no Claude Code. Não precisa preencher nada — o Claude vai te pedir as informações necessárias durante a instalação e depois configurar tudo automaticamente.

📋
Tenha em mãos: seu endereço Gmail e a App Password de 16 caracteres gerada no Passo 2. O Claude vai pedir os dois antes de começar.
Prompt — Cole no Claude Code
Configure o Prep Call Agent para mim. Antes de executar qualquer passo, faca as seguintes perguntas e aguarde minha resposta para cada uma: Pergunta 1: "Qual e o seu endereco Gmail que vai enviar os briefings?" Pergunta 2: "Qual e a sua App Password de 16 caracteres gerada no Passo 2? (sem espacos, ex: abcdabcdabcdabcd)" Guarde as respostas como [EMAIL] e [APP_PASSWORD]. So prossiga apos receber as duas respostas. --- Apos receber as respostas, execute os passos abaixo em ordem: PASSO A - Detecte o caminho absoluto da pasta atual via Bash: Execute: pwd Guarde o resultado como [PASTA]. Use esse valor em todos os caminhos de arquivo dos proximos passos. PASSO B - Instale as dependencias via Bash: npm init -y && npm install nodemailer PASSO C - Crie o arquivo [PASTA]/send-email.js com o seguinte conteudo, substituindo [EMAIL] e [APP_PASSWORD] pelas respostas do usuario: const nodemailer = require('nodemailer'); const fs = require('fs'); const [,, to, subject, bodyFile] = process.argv; if (!to || !subject || !bodyFile) { console.error('Uso: node send-email.js <to> <subject> <bodyFile>'); process.exit(1); } if (!fs.existsSync(bodyFile)) { console.error('Arquivo nao encontrado:', bodyFile); process.exit(1); } const body = fs.readFileSync(bodyFile, 'utf8'); const transporter = nodemailer.createTransport({ service: 'gmail', auth: { user: '[EMAIL]', pass: '[APP_PASSWORD]' } }); transporter.sendMail( { from: '[EMAIL]', to, subject, text: body }, (err, info) => { if (err) { console.error('Erro:', err.message); process.exit(1); } console.log('Enviado:', info.messageId); } ); PASSO D - Crie o arquivo [PASTA]/agent-prompt.md com o seguinte conteudo exato, substituindo todas as ocorrencias de [PASTA] pelo caminho detectado no Passo A: Voce e um agente de preparacao de reunioes. Execute os passos abaixo em ordem, de forma completa e autonoma. Nunca invente informacoes. Se nao encontrar um dado, escreva "nao encontrado" na secao correspondente. ## PASSO 1 - CALCULAR DATA DE AMANHA E LER AGENDA Calcule a data de amanha no timezone America/Sao_Paulo (BRT, UTC-3). Use gcal_list_events com: - timeMin: amanha 00:00:00 RFC3339 sem timezone (ex: "2025-04-09T00:00:00") - timeMax: amanha 23:59:59 RFC3339 sem timezone (ex: "2025-04-09T23:59:59") - timeZone: "America/Sao_Paulo" - condenseEventDetails: false Extraia: titulo (summary), horario (start), participantes (attendees), organizador (organizer) — o organizador e o REP. ## PASSO 2 - FILTRAR EVENTOS ELEGIVEIS Para cada evento: 1. Dominio do organizador = dominio interno 2. Ignore dominios pessoais: gmail.com, googlemail.com, hotmail.com, hotmail.com.br, outlook.com, outlook.com.br, yahoo.com, yahoo.com.br, icloud.com, live.com, msn.com, uol.com.br, bol.com.br, terra.com.br 3. Ignore participantes com mesmo dominio do organizador 4. Se restar ao menos 1 externo corporativo: evento elegivel 5. Agrupe externos por dominio (dominio = empresa) Se nenhum elegivel: encerre silenciosamente, sem enviar nada. ## PASSO 3 - RESEARCH REGRA CRITICA: identificador da empresa = dominio exato do email. Nunca use dados de outra empresa com nome similar. Para cada empresa (dominio externo): Etapa 0 - WebFetch em https://[dominio] (obrigatorio, sempre primeiro). Se nao carregar, tente https://www.[dominio]. Busca 1: "[empresa]" noticias 2024 2025 (ou "[company]" news 2024 2025) Busca 2: "[empresa]" contratacoes demissoes produto investimento (ou "[company]" funding hiring layoffs product launch) Tente LinkedIn da empresa se acessivel sem login. Busca 3: "[empresa]" modelo de negocio mercado segmento (ou "[company]" business model market segment) Para cada participante externo: Busca 4: "[nome completo]" "[empresa]" LinkedIn Busca 5: "[nome completo]" "[empresa]" entrevista artigo post palestra (ou "[full name]" "[company]" interview article post talk) Regra: sem dado real, escreva "nao encontrado". Nunca invente. ## PASSO 4 - GERAR O BRIEF Regras: - Um brief por empresa (2 empresas no mesmo evento = 2 briefs) - Nunca inventar. "nao encontrado" quando sem dado real - Hipoteses de dor SEMPRE marcadas como hipoteses - Observacao de abertura baseada em sinal real, nunca generica - Brief sempre em portugues Formato exato: MEETING BRIEF - [Empresa] | [DD/MM/AAAA] | [HH:MM BRT] ---------------------------------------- PARTICIPANTES - [Nome] | [Cargo] | LinkedIn: [url ou "nao encontrado"] ---------------------------------------- SITUACAO ATUAL DA EMPRESA [baseado em pesquisa real - ou "nao encontrado"] ---------------------------------------- O QUE ESSE CARGO PROVAVELMENTE PRIORIZA [inferencia baseada no cargo e contexto - nao e fato] ---------------------------------------- HIPOTESES DE DOR - sao hipoteses, nao fatos 1. [dor 1 - baseada em contexto real] 2. [dor 2 - baseada em contexto real] 3. [dor 3 - baseada em contexto real] ---------------------------------------- OBSERVACAO DE ABERTURA [baseada em sinal real - ou "nao encontrado - evitar generica"] ---------------------------------------- PERGUNTAS DE DISCOVERY 1. [pergunta que testa hipotese de dor 1] 2. [pergunta que testa hipotese de dor 2] 3. [pergunta sobre prioridades ou contexto de decisao] ---------------------------------------- Fontes consultadas: - [url 1] - [url 2] ## PASSO 5 - ENVIAR POR EMAIL Para cada brief gerado: 5a. Salve o brief usando a ferramenta Write: Caminho: [PASTA]/brief-temp.txt Conteudo: o brief completo em texto simples 5b. Envie via Bash: node "[PASTA]/send-email.js" "[email do rep]" "[PREP] [Empresa] - [DD/MM] [HH:MM]" "[PASTA]/brief-temp.txt" 5c. Delete o arquivo temporario via Bash: rm "[PASTA]/brief-temp.txt" Repita 5a, 5b, 5c para cada empresa por evento elegivel. Apos enviar todos os emails, encerre. --- PASSO E - Leia o arquivo [PASTA]/agent-prompt.md que acabou de criar e use o conteudo completo como valor do parametro prompt ao criar um scheduled task com: - taskId: prep-call-agent - cronExpression: 0 18 * * * - description: Le agenda do dia seguinte e envia briefings as 18h BRT - notifyOnCompletion: true Confirme quando todos os passos estiverem concluidos.
⚡
O Claude vai fazer 2 perguntas antes de começar — responda e deixe rodar. A instalação completa leva cerca de 1–2 minutos.
